# TsaoSciComputation v3.0.1

This maintenance release converts the verified 3.0.0 repository into a governed, attestable release line without changing its scientific claim boundary.

## Reliability and fail-closed behavior

- Prevented abnormal, failed, incomplete, or negatively converged solver output from being promoted through positive substrings.
- Required completed execution before convergence can be asserted.
- Rejected unknown calculation-contract fields, malformed mappings, non-string field names, invalid tolerances, and non-finite physical-validation inputs.
- Required structured human-approval evidence for scientific acceptance.
- Corrected controlled state-transition mutation probes and retained the 64/64 mutation gate.

## Packaging and supply chain

- Made `VERSION` the single authoritative package version source.
- Added automatic version synchronization checks for package metadata, runtime `__version__`, README files, `CITATION.cff`, and the Changelog.
- Added deterministic SPDX 2.3 and CycloneDX 1.6 SBOM generation.
- Added a complete release Manifest and SHA-256 checksum set.
- Added GitHub/Sigstore build-provenance and SBOM attestations.
- Added formal GitHub Release publication with reproducible ZIP, tar.gz, and Wheel artifacts.

## Governance and security

- Expanded the private vulnerability reporting and supported-version policy.
- Aligned contribution guidance with the canonical single-`main` repository policy: external contributions use forks; the upstream repository does not retain feature branches.
- Kept benchmarks separate from deterministic release acceptance.

## Verified boundary

The release validates orchestration, contracts, routing, deterministic fixtures, conservative parsing, numerical and physical validation gates, provenance, security controls, reproducible packaging, SBOM generation, and isolated installation. It does not claim that third-party scientific solvers, commercial licenses, production HPC environments, or plant control systems were live-tested or certified.
