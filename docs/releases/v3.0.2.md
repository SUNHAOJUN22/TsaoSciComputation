# TsaoSciComputation v3.0.2

This maintenance release deepens long-term repository governance without creating upstream feature branches.

## Highlights

- Adds a machine-readable, fail-closed C0–C5 result-confidence model that separates completion, convergence, physical validation, bounded research evidence, expert acceptance, and engineering-decision readiness.
- Adds a pinned, branchless weekly dependency-vulnerability audit with read-only permissions and retained JSON evidence.
- Adds eight deterministic scientific reference benchmarks with analytical, conservation, and invariant acceptance criteria.
- Adds strict Adapter Schema validation and A0–A5 certification; A5 is reserved for versioned live-solver evidence.
- Adds critical coverage, compatibility, and repository Ruleset desired-state policies.
- Adds CODEOWNERS coverage for security-sensitive, release, contract, adapter, state, and validation paths.
- Adds structured software-defect, scientific-validity, and capability-request issue forms while disabling unsafe blank issues.
- Adds support, conduct, and dependency-maintenance policies aligned with the single-`main` upstream model.
- Adds regression tests for ownership, issue routing, pinned maintenance Actions, dependency-audit permissions, and the absence of branch-creating Dependabot configuration.

## Verification

The release is accepted only after Python 3.10 core gates, Python 3.13 deterministic release gates, dependency audit, benchmark telemetry, Skill installation roundtrip, single-main verification, reproducible source and Wheel builds, SBOM generation, release checksums, and GitHub/Sigstore attestations.

## Trust boundary

The dependency audit detects known Python-package vulnerabilities. It does not certify third-party solvers, scientific models, commercial licenses, production HPC systems, plant controls, or user-supplied data.
